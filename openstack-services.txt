Git RepositoriesOpenStack services (categorized) — verified against OpenStack 2025.2 “Flamingo” (released Oct 1, 2025)

IaaS (core infrastructure):
- Identity & access: Keystone
- Images: Glance
- Resource placement / inventory: Placement
- Compute (VMs): Nova
- Bare metal: Ironic
- Accelerators (GPU/FPGA attach): Cyborg
- Networking (L2/L3, routers, SGs, IPAM): Neutron
- Load balancing: Octavia
- DNS-as-a-service: Designate
- Storage (Object): Swift
- Storage (Block): Cinder
- Storage (Shared file): Manila
- Backup / DR: Freezer, Freezer-api

PaaS (platform, orchestration, and day-2 operations):
- Orchestration (templates/stacks): Heat
- Workflow / automation: Mistral
- DBaaS: Trove
- Messaging / queues: Zaqar
- NFV orchestration: Tacker
- Key management / secrets: Barbican
- Telemetry (metering/metrics collection): Ceilometer
- Telemetry (alarming): Aodh
- Observability integration (Prometheus multi-tenancy + Keystone auth): Aetos
- Log management: Venus
- Resource reservations: Blazar
- Rating / chargeback: Cloudkitty
- Infrastructure optimization: Watcher
- RCA / event correlation: Vitrage
- Instance high availability: Masakari
- Compute inside Object Storage (run code near Swift): Storlets
- Day-2 operations / self-service workflows: Adjutant

FaaS (Serverless):
- No core OpenStack “functions-as-a-service” in the 2025.2 service set; typically delivered via Kubernetes serverless layers deployed on top of OpenStack (e.g., Knative/OpenFaaS).

CaaS (Containers):
- Kubernetes / container infrastructure management: Magnum
- Containers-as-a-service (run containers as first-class resources): Zun

SaaS (end-user / UI):
- Classic dashboard: Horizon
- Modern dashboard: Skyline (Skyline-apiserver + Skyline-console)

Practical caveat:
- Even if a project is in the 2025.2 service set, packaging can vary by distribution/deployer; for example, Kolla notes it dropped building Venus container images due to the project’s inactivity and related Horizon plugin build issues.
